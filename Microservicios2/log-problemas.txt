PROBLEMAS
1 - FEIGN -> No se crea la bean.
	SOLUCIÓN: @ImportAutoConfiguration (No funciono)
	SOLUCIÓN: pom -> version de Spring ->2.4.5 a 2.3.4.RELEASE (Funciono)
2 - Configuración de VS Code para realizar más de una instancia
	SOLUCIÓN: Run -> OpenConfigurations ->launch.json
	Agregar configuraciones especificando puerto:
	{
            "type": "java",
            "name": "Launch (8081) ServicioProductosApplication",
            "request": "launch",
            "mainClass": "ayi.microservicios.servicioproductos.ServicioProductosApplication",
            "projectName": "servicio-productos",
            "vmArgs": "-Dserver.port=8081"
        }
	Para ejecutarlas usar la opcion de la barra izquierda Run and Debug. 	Elegir la instancia que se quiere runnear y darle al run.
	(Funciona)
3- @Value no trae valor especificado de application.properties 
	SOLUCIÓN: @PropertySource(value="application.properties")
4- org.springframework.cloud.client.discovery.health.DiscoveryCompositeHealthIndicator no existe
	SOLUCIÓN: cambiar versión de spring boot 2.3.4 -> 2.2.2.RELEASE (No funciono)
	SOLUCIÓN: https://stackoverflow.com/questions/58546540/upgraded-spring-boot-from-2-1-9-to-2-2-0-now-getting-exception-while-starting
	Incompatibilidad spring boot - spring cloud
	cambiar version spring cloud -> Greenwich -> Hoxton (Funciono)


5- Zuulfilter Exception
